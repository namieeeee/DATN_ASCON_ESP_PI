Samples

参考类型

每个IP的LL Sample根据类型分类，每个分类对应一个Sample Project

  • Polling: 使用轮询模式进行操作，阻塞等待函数完成
  • Interrupt: 使用中断模式进行操作，非阻塞，完成后会进入相应的回调函数
  • DMA: 使用DMA模式进行操作，需要配置DMA通道，非阻塞，完成后会进入相应的回调函数

测试案例

每个Sample会提供多种TestCase，每次运行Sample只会执行一个TestCase，用户可以自行选择适合的TestCase进行测试
TestCase可以在init.h中进行配置，如I2C：


  • I2C_MASTER_TX: 作为主机发送数据
  • I2C_MASTER_RX: 作为主机接收数据
  • I2C_SLAVE_TX: 作为从机发送数据
  • I2C_SLAVE_RX: 作为从机发送数据

main.c中会根据所配置的TestCase执行对应的函数，如选择 I2C_MASTER_TX ，则调用 i2c0_master_transmit_it()

外设实例

Sample默认使用外设的实例0进行操作，如SPI，则使用 SPI0 ，相关的函数在 spi0.c中，以 sip0_ 开头
Sample为所有实例都提供了对应的代码，想要使用其它实例，如SPI1，则需要修改相关代码:

  • 将init.h中的文件引用 "spi0.h" 修改为 "spi1.h"
  • 将init.c中传入Init函数的寄存器指针 SN_SPI0 修改为 SN_SPI1
  • 将init.c中init函数内 SPI0的时钟和中断设定 修改为 SPI1的时钟和中断设定
    • 如 LL_RCC_Enable_SSP0CLK() 修改为 LL_RCC_Enable_SSP1CLK()
    • 如 NVIC_EnableIRQ(SPI0_IRQn) 修改为 NVIC_EnableIRQ(SPI1_IRQn)
  • 将init.c中init函数内 SPI0的引脚设定 修改为 SPI1的引脚设定
    • 如 LL_GPIO_SetAFIO_P00(LL_GPIO_P00_SPI_SCK0) 修改为 LL_GPIO_SetAFIO_P01(LL_GPIO_P01_SPI_SCK1)
  • 将main.c, it.c中调用的 sip0_ 开头的函数修改为 sip1_ 开头
    • 如 spi0_master_transmit_it 修改为 spi1_master_transmit_it
