TPM Interface » Usage Flow

说明

在 OC 使用中，可以将 Channel 作为定时器、EMC 或者 Force:

  1. 定时器( LL_TPM_OCMODE_TIMER): 该 Channel 只作为计数，没有任何输出
  2. EMC:
    • Low LL_TPM_OCMODE_LOW): 当 TC == MR，EMC 输出 低电平
    • High( LL_TPM_OCMODE_HIGH): 当 TC == MR，EMC 输出 高电平
    • Toggle ( LL_TPM_OCMODE_TOGGLE): 当 TC == MR，EMC 输出 电平反向
  3. Force:
    • Low( LL_TPM_OCMODE_FORCED_LOW): 强制输出 低电平
    • High( LL_TPM_OCMODE_FORCED_HIGH): 强制输出 高电平

TPM OC有三种用法：Base，Interrupt 和 DMA

  1. Base：只根据 LL_TPM_OC_ConfigChannel 中的设定进行相应的操作
  2. Interrupt：支援 Base 功能，此外，当TC == MRn 时触发 Interrupt
    • 若 cm 为 LL_TPMx_CM_COUNTING_CENTER_DOWN (x = 0, 125) 时只在TC 递减 时触发 Interrupt
    • 若 cm 为 LL_TPMx_CM_COUNTING_CENTER_UP (x = 0, 125) 时只在TC 递增 时触发 Interrupt
  3. DMA：支援 Base 功能，此外，将 Memory 的值通过 DMA 发送给 MRn
    • 若 cm 为 LL_TPMx_CM_COUNTING_CENTER_DOWN (x = 0, 125) 时值会在TC 递减 且 TC == MR 后将 Memory 的值通过 DMA 发送给 MRn
    • 若 cm 为 LL_TPMx_CM_COUNTING_CENTER_UP (x = 0, 125) 时值会在TC 递增 且 TC == MR 后将 Memory 的值通过 DMA 发送给 MRn
    • 其余情况当 TC == MR 后，将 Memory 的值通过 DMA 发送给 MRn
    • 在 Memory 的值通过 DMA 发送给 MRn 后根据 LL_TPM_ConfigLoadMode 设定在特定时刻才会实际生效，若不支援加载模式的 Channel 会在 TC == 0 时生效
