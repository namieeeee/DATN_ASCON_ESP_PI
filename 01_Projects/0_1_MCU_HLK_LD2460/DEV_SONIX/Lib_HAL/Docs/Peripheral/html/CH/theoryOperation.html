Theory of Operation

该章节介绍了HAL驱动的常规操作。说明了Common Peripheral Functions 在所有的HAL驱动中以及Function Call Sequence 中是共用的。 Data Transfer Functions 介绍了对外设的数据读取/写入操作是如何进行的。

Common Peripheral Functions

各个HAL驱动包含下面的函数：

  • GetVersion: 任何时间都可以调用，用于获取驱动接口的版本信息。
  • HAL_IPx_RegisterCallback: 任何时间都可以调用，需要的时候，注册IPx_Callback_T 回调函数。
  • HAL_IPx_Init: 该函数执行下面的操作：
    • 设置外设Instance以及需要的处理程序(Handler)数据，在IPx_Handle_T 的init 栏有定义。
    • 在HAL_IPx_MspInit弱函数中分配I/O资源，该函数内置于HAL_IPx_Init
    • 使用HAL_IPx_RegisterCallback API注册一个可选的IPx_Callback_T 回调函数。
  • IPx_Callback_T: 是一个可选的回调函数，是使用HAL_IPx_RegisterCallback 函数注册的。 各个外设IP定义了它自己的回调ID，各个回调ID仅引用一个回调函数。如果需要用注册回调函数替换弱MSPInit函数， 应该在HAL_IPx_Init以前替换。 由中断服务程序发起调用函数，表示硬件事件或者Data Block传输操作的完成。
  • HAL_IPx_UnRegisterCallback: 如需取消注册回调函数，请参考HAL_IPx_RegisterCallback中的函数， 通过回调函数ID释放。
  • HAL_IPx_DeInit: 如需初始化，请参考HAL_IPx_Init中的函数。释放接口占用的I/O资源。

Function Call Sequence 章节对各个函数的操作作了更多的说明。 各个驱动接口会有更多特有的函数，在各驱动单独的章节中会对这些函数进行说明。

Function Call Sequence 章节对各个函数的操作作了更多的说明。 各个驱动接口会有更多特有的函数，在各驱动单独的章节中会对这些函数进行说明。

Callback Functions

各个HAL驱动可能支持多个回调函数。这些回调函数有两个访问方式。

  • 通过弱函数
    各个回调函数默认会有一个弱原型函数。通过在用户
    工程修改函数，禁用默认的函数。

  • 通过注册回调函数
    除了修改弱函数，通过RegisterCallBack API修改用户自定义函数。用户自定义的
    回调函数的优先级会高于默认的弱函数。
    对于注册回调函数，configUSE_IPx_REGISTER_CALLBACKS 需要设置为1

Function Call Sequence

为了确保驱动的正常运行，API函数诸如GetVersion, HAL_IPx_RegisterCallback, HAL_IPx_Init, HAL_IPx_UnRegisterCallback, HAL_IPx_DeInit 需要按照下面的顺序调用：

GetVersion 函数可以在任何时候调用，以获取驱动需要的信息。 这些函数总是返回相同的信息。

Start Sequence

开始处理某个外设的时候，函数诸如Initialize 需要按照下面的顺序调用：

HAL_IPx_RegisterCallback (...); // 可选的，注册MSPInit/MSPDeInit回调函数
HAL_IPx_Init (...); // 外设通电，设置IRQ/DMA/AFIO/时钟
  • HAL_IPx_RegisterCallback, 为了用自定义函数MSPInit/MSBDeInit替换默认弱函数MSPInit/MSBDeInt 应该在HAL_IPx_Init 以前替换; 在HAL_IPx_Init 以后，需要的话可以注册所有的回调函数。
  • HAL_IPx_Init(SONIX_POWER_FULL) 设置外设寄存器，包括中断(NVIC) 和DMA(可选的)。 函数可以多次调用；如果寄存器已经设置了，不会执行任何的操作，会返回HAL_OK 。

Stop Sequence

停止处理外设的时候，函数诸如De-initialize 需要按照下面的顺序调用：

HAL_IPx_UnRegisterCallback (SONIX_POWER_OFF); // 可选的，取消登记回调函数。
HAL_IPx_DeInit (...); // 任何挂起的传输，重置 IRQ/DMA/时钟, 外设电源关闭, 释放I/O引脚

HAL_IPx_UnRegisterCallback 和HAL_IPx_DeInit 函数总是执行，并用于将外设置于一个安全状态,
例如在发生数据发送错误以后。如果需要在错误条件重启外设，应首先执行
Stop Sequence ，然后执行Start Sequence 。

  • HAL_IPx_UnRegisterCallback , 如果不适用任何之前登记的回调函数，我们可以调用该API，取消注册回调函数。
  • HAL_IPx_DeInit 总是清除任何的运行时资源。

I/O Pins operator

通常，在MSPInit函数，应考虑到I/O引脚资源操作，并在MSPDeInit函数中释放。
对于指定的板子，他们有自己的外设引脚配置。因此，对板子的外设的AFIO设置不应再次配置。

Data Transfer Functions

某个HAL驱动运行 非阻塞(驱动配置对外设的读写访问，并立刻返回至调用应用) 和阻塞函数，以传输数据到外设。
在阻塞 模式，函数名不会有任何的结束标志。
在非阻塞 模式，函数名会以下面的结束标志：

  • _IT 代表中断操作。
  • _DMA 代表DMA操作。

在非阻塞 数据传输中，应用可使用名为HAL_IPx_GetState的函数查询传输状态。
数据传输完成以后，驱动调用某个特定的回调函数。在和外设进行数据传输的时候，
应用可以决定：

  • 等待(使用一个RTOS排程器)某个回调事件完成。RTOS由让驱动独立于RTOS的应用代码控制。
  • 使用返回传输数据笔数的Polling函数，显示进度，部分读取或者填充数据传输缓冲。
  • 对于下一次数据传输，准备另外一个数据传输缓冲。

Peripheral Configuration

对于某个设备系列，驱动是可以配置的。配置选项存储在某个Central文件，名为
\ sn34f78x_hal_conf.h 。 sn34f78x_hal_conf.h 定义了外设函数开关，使用该函数开关裁减用户代码大小。
以I2C为例，列出了如下的内容。

用户必须 从SDK/HAL/Conf 文件夹复制sn34f78x_hal_conf.h至应用工程。
以I2C为例，配置文件清单为：

#define configUSE_I2C_PERIPHERAL 0 /**< 0: 禁用I2C驱动; 1: 使能I2C驱动; 默认是0 */
#if (configUSE_I2C_PERIPHERAL == 1)
// <e> 使能回调
#define configUSE_I2C_REGISTER_CALLBACKS 0 /**< 0: 禁用回调API; 1: 使能回调API; 默认是0 */
// </e> 使能回调
// <e> 使能阻塞API
#define configUSE_I2C_BLOCKING 0 /**< 0: 禁用阻塞API; 1: 使能阻塞API; 默认是0 */
// </e> 使能阻塞API
// <e> 使能IT API
#define configUSE_I2C_NONBLOCKING_IT 0 /**< 0: 禁用IT API; 1: 使能IT API; 默认是0 */
// </e> 使能IT API
// <e> 使能BURST API
#define configUSE_I2C_NONBLOCKING_BURST 0 /**< 0: 禁用BURST API; 1: 使能BURST API; 默认是0 */
// </e> 使能BURST API
#include "hal_i2c.h"
#endif

Code Example

下面的范例代码展示了I2C HAL驱动的工作流。

I2C Blocking Code :

#include "sn34f78x_hal.h"
/*****************************************************************************/
/* data buffer size */
#define DATA_LEN 32
/* data buffer for transmitting data */
static uint8_t data_tx[DATA_LEN];
/* data buffer for receiving data */
static uint8_t data_rx[DATA_LEN];
/*****************************************************************************/
I2C_Handle_T i2c_tx_handle = {
/* link to I2C1 IP */
.instance = SN_I2C1,
.init = {
.addressing_mode = I2C_ADDRESSINGMODE_7BIT,
.clock_speed = I2C_CLOCKSPEED_FAST_PLUS,
.gc_en = HAL_DISABLE,
.own_address = 0x7f,
},
};
static int32_t _i2c_blocking_transmit(void)
{
if (HAL_I2C_Init(&i2c_tx_handle) != HAL_OK)
{
return HAL_ERROR;
}
// timeout = 6000ms
if (HAL_I2C_Master_Transmit(&i2c_tx_handle, i2c_tx_handle.init.own_address, data_tx, DATA_LEN, 6000) != HAL_OK)
{
return HAL_ERROR;
}
if (HAL_I2C_DeInit(&i2c_tx_handle) != HAL_OK)
{
return HAL_ERROR;
}
return HAL_OK;
}
/*****************************************************************************/
I2C_Handle_T i2c_rx_handle = {
/* link to I2C2 IP */
.instance = SN_I2C2,
.init = {
.addressing_mode = I2C_ADDRESSINGMODE_7BIT,
.clock_speed = I2C_CLOCKSPEED_FAST_PLUS,
.gc_en = HAL_DISABLE,
.own_address = 0x7f,
}};
static int32_t _i2c_blocking_receive(void)
{
if (HAL_I2C_Init(&i2c_rx_handle) != HAL_OK)
{
return HAL_ERROR;
}
// timeout = 6000ms
if (HAL_I2C_Slave_Receive(&i2c_rx_handle, data_rx, DATA_LEN, 6000) != HAL_OK)
{
return HAL_ERROR;
}
if (HAL_I2C_DeInit(&i2c_rx_handle) != HAL_OK)
{
return HAL_ERROR;
}
return HAL_OK;
}
/*****************************************************************************/
int32_t main(void)
{
_i2c_blocking_transmit();
_i2c_blocking_receive();
while (true)
{
__NOP();
}
}

I2C Non-Blocking Code :

#include "sn34f78x_hal.h"
/*****************************************************************************/
/* data buffer size */
#define DATA_LEN 32
/* data buffer for transmitting data */
static uint8_t data_tx[DATA_LEN];
/* data buffer for receiving data */
static uint8_t data_rx[DATA_LEN];
/* tx complete flag */
static __IO bool tx_flag = false;
/* rx complete flag */
static __IO bool rx_flag = false;
/*****************************************************************************/
void HAL_I2C_MasterTxCpltCallback(I2C_Handle_T *hi2c)
{
tx_flag = true;
}
I2C_Handle_T i2c_tx_handle = {
/* link to I2C1 IP */
.instance = SN_I2C1,
.init = {
.addressing_mode = I2C_ADDRESSINGMODE_7BIT,
.clock_speed = I2C_CLOCKSPEED_FAST_PLUS,
.gc_en = HAL_DISABLE,
.own_address = 0x7f,
},
};
static int32_t _i2c_interrupt_transmit(void)
{
if (HAL_I2C_Init(&i2c_tx_handle) != HAL_OK)
{
return HAL_ERROR;
}
if (HAL_I2C_Master_Transmit_IT(&i2c_tx_handle, i2c_tx_handle.init.own_address, data_tx, DATA_LEN) != HAL_OK)
{
return HAL_ERROR;
}
while (tx_flag == false)
{
__NOP();
}

return HAL_ERROR;
}
return HAL_OK;
}

{
return HAL_ERROR;
}
while (rx_flag == false)
{
__NOP();
}
if (HAL_I2C_DeInit(&i2c_rx_handle) != HAL_OK)
{
return HAL_ERROR;
}
return HAL_OK;
}
/*****************************************************************************/
int32_t main(void)
{
_i2c_interrupt_transmit();
_i2c_interrupt_receive();
while (true)
{
__NOP();
}
}
