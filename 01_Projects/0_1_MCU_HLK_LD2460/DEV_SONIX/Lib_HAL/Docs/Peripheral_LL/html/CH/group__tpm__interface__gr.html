Description

TPM 可使用内部时钟或者外部输入信号边沿变化进行计数，并且可以根据设置 Match register 的值以定时选择是否生成中断或执行其他操作
TPM 还支持 Cpature 功能，用于在输入信号边沿变化时捕获此时的计数值，可选择是否生成中断


特点

  1. Six 16-bit counter/timers with a programmable 8-bit prescaler.
  2. Two 16-bit timers with a programmable 8-bit prescaler. (CT16B6/CT167)
  3. Counter or timer operation
  4. Six 16-bit capture channels that can take a snapshot of the timer value when an input signal transitions. A capture event may also optionally generate an interrupt
  5. The timer and prescaler may be configured to be cleared on a designated capture event. This feature permits easy pulse-width measurement by clearing the timer on the leading edge of an input pulse and capturing the timer value on the trailing edge
  6. Eight 16-bit match registers that allow: – Continuous operation with optional interrupt generation on match. – Stop timer on match with optional interrupt generation. – Reset timer on match with optional interrupt generation.
  7. Up to 4 (CT16B0, CT16B2, CT16B5, CT16B1), or 2 (CT16B3, CT16B4) PWM outputs corresponding to match registers with the following capabilities: – Set LOW on match. – Set HIGH on match. – Toggle on match. – Do nothing on match.
  8. For CT16B0, up to 4 inverse waveform of the PWM signals, and builds in programmable dead-band function.
  9. For CT16B3 and CT16B4, up to 2 complete inverse waveform of the PWM signals, and builds in programmable dead-band function
  10. Break function for triggers to stop PWM. (CT16B0)
  11. Supply DMA transfer


功能

不同TPM支持不同的功能，如下：
CT16B0

特点 功能 说明 注意事项
计数来源 — 通过 LL_TPM_Init API , 设置 LL_TPM_Init_T 的 "ctm" 域 "mode" 可以设置为 CT16B0 Counter Time Mode ，支援内部时钟或者外部输入信号作为计数来源
计数模式(递增或递减) 通过 LL_TPM_Init API , 设置 LL_TPM_Init_T 的 "cm" 域 "cm" 可以设置为 CT16B0 Counting Mode
Capture 通过 LL_TPM_CAP_Config API 请参考： LL_TPM_Capture_T , 该功能只允许在 LL_TPM_Init_T 的 "mode" 设置为 LL_TPM0_CTM_TIMER 时使用
Break Pin 通过 LL_TPM_ConfigBreak API 请参考： LL_TPM_Break_T ， 通过 Break Pin 信号关闭 TPM 的 PWM Pin 输出
Channel 0 ~ 3 PWM 通过 LL_TPM_PWM_ConfigChannel API , 设置 LL_TPM_OC_T 的 "mode" 字段 "mode" 设置为： LL_TPM_OCMODE_PWM1 或 LL_TPM_OCMODE_PWM2
PWMN 通过 LL_TPM_PWM_ConfigPWMN API 请参考： LL_TPM_PWMN_T ， 该功能只能在对应 channel 设置为 PWM 时使用
定时器 通过 LL_TPM_OC_ConfigChannel API , 设置 LL_TPM_OC_T 的 "mode" 字段 "mode" 设置为 LL_TPM_OCMODE_TIMER
EMC "mode" 设置为： LL_TPM_OCMODE_LOW 或 LL_TPM_OCMODE_HIGH 或 LL_TPM_OCMODE_TOGGLE
Force "mode" 设置为： LL_TPM_OCMODE_FORCED_LOW 或 LL_TPM_OCMODE_FORCED_HIGH
加载模式 通过 LL_TPMx_SetTCMatchValueToMRn (x=0,125,34,8) 与LL_TPM67_SetTCMatchValueToMR0 API 请参考： LL_TPM_LoadMode_T ， 该功能决定当 TPM run 之后，输入数据修改原来的 Match register 值后，何时才会真正生效。一般用在 DMA 传输 和 LL_TPMx_SetTCMatchValueToMRn (x=0,125,34,8) 与LL_TPM67_SetTCMatchValueToMR0 之前
动态更改 Match register 通过 LL_TPM0_SetTCMatchValueToMRn API 当 TPM run 之后，更改其 Match register 的值，其更改的值真正生效时机依赖 LL_TPMx_SetTCMatchValueToMRn (x=0,125,34,8) 与LL_TPM67_SetTCMatchValueToMR0
Channel 9 定时器 通过 LL_TPM_OC_ConfigChannel API , 设置 LL_TPM_OC_T 的 "mode" 字段,
例如 LL_TPM_OCMODE_TIMER
