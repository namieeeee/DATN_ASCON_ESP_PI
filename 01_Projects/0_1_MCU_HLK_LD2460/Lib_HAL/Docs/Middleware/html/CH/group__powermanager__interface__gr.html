Description

PowerManager 电源控制模块，将对PWR、RTC、GPIO 功能模块化，操作简单，减少产品开发周期。 用于程序 Run-Time 时，通过进入不同的低功耗模式，完成不同程度的省电功能。共有Sleep、DeepSleep、 DeepPowerDown三种低功耗模式，省电能力从低到高。并且提供两种唤醒低功耗模式的方式，分别 是RTC ALARM 中断唤醒以及GPIO 中断唤醒，两种唤醒方式可同时共存


  • 特点
  1. 支持 Sleep、DeepSleep、DeepPowerDown 三种低功耗模式
  2. 支持 RTC ALARM 和 GPIO 两种唤醒方式
  3. 支持访问BackUp SARM和BackUp REG
  4. 支持在程序运行时主动改变系统时钟


  • 配置设置 (sn34f78x_hal_conf.h)
  1. 需要打开 configUSE_POWERMANAGER_MIDDLEWARE
  2. 如果要使用 GPIO 唤醒低功耗模式，需要设定 configUSE_POWERMANAGER_GPIO_WAKE_UP = 1
  3. 如果要使用 RTC 唤醒低功耗模式，需要设定 configUSE_POWERMANAGER_RTC_WAKE_UP = 1


  • 功能
    PowerManager 支持下面的功能：
    模式 说明 注意事项
    Sleep 低功耗模式 进入Sleep 模式，系统时钟停止，MCU暂停处理命令，系统功耗略微降低，可通过任意中断唤醒 Sleep 模式下，系统可通过任何中断将其唤醒。唤醒后系统从睡眠时代码段继续运行
    DeepSleep低功耗模式 进入DeepSleep 模式，系统时钟停止，MCU暂停处理命令，系统功耗降低，可通过特定的唤醒源唤醒 DeepSleep 模式下，系统可通过特点的唤醒源将其唤醒，包括GPIO、RTC、ETH、USB 。唤醒后系统从睡眠时代码段继续运行
    DeepPowerDown低功耗模式 进入DeepPowerDown 模式，系统时钟停止，MCU停止处理命令，系统功耗大幅度降低，可通过特定的唤醒源唤醒 DeepPowerDown 模式下，系统可通过特点的唤醒源将其唤醒，包括GPIO、RTC。唤醒后系统重启

PowerManager WakeUp Source

通过模块 PowerManager ，用户可以使用任意中断唤醒Sleep模式，以及使用特定的唤醒源来唤醒DeepSleep和DeepPowerDown模式， 特定设置唤醒源清单如下：

唤醒源 说明
RTC 配置 RTC 为唤醒源，使用 RTC ALARM配置闹钟时间，可配置绝对时间和相对时间,
到达对应的闹钟时间时，触发ALARM中断，会唤醒低功耗模式
GPIO 配置具体 GPIO 为唤醒源，配置 该GPIO 中断，当中断触发时，会唤醒低功耗模式

PowerManager_backup BackUp

PowerManager 为用户提供 BKP_SRAM 和 BKP_REG 寄存器的操作接口。 1、备用SRAM(BKP_SRAM)大小为4KB， 通过 HAL_PowerManager_BKUPSRAM_Write 和 HAL_PowerManager_BKUPSRAM_Read 接口完成读写操作 2、备用寄存器(BKP_REG)共有20个，通过 HAL_PowerManager_BKUPREG_Write 和 HAL_PowerManager_BKUPREG_Read 接口完成读写操作

PowerManager SysCLK :

为了更好的减少系统的能耗， PowerManager 为用户提供了选择不同频率的系统时钟的接口，系统时钟越低，系统所需要的能耗也就越低。

  1. 目前提供了5中常用的系统时钟频率供用户选择，分别为12Mhz，24Mhz，48Mhz，96Mhz，192Mhz.
  2. 为了减少系统的能耗，建议进入低功耗Sleep 前，可将系统时钟调低。
