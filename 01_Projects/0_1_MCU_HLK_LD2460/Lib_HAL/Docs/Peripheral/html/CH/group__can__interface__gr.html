Description

控制器局域网(CAN bus)是一种功能丰富的车用总线，被设计用于在不需要主机（Host）的情况下，允许网络上的单片机和设备相互通信。
它基于消息传递协议，设计之初在车辆上采用复用通信线缆，以降低铜线使用量，后来也被其他行业所使用。
对于每个设备，以串行方式进行数据帧的传输，采用的方式为：如果有一个以上的设备同时发送, 具有最高优先级的设备进行传输，其他设备需要停止。所有的设备都会收到数据帧，包括发送设备。
更多的信息，可以查看维基百科： Controller Area Network.

  • Features
  1. Supports CAN 2.0A,B
  2. Data rates up to 1Mbit/s
  3. 15 receive buffers (RB) ­ * - Operation in FIFO ­ * - Received frames which are “not accepted” or “incorrect” don’t overwrite already stored frames.
  4. Two transmit buffers ­ * - 1 frame slots of primary transmit buffer (PTB) ­ * - 7 frame slots of secondary transmit buffer (STB) ­ * - Operation in FIFO or priority decision mode
  5. 16 sets of internal acceptance filter
  6. Extended features ­ * - Single shot transmission mode ­ * - Listen only mode ­ * - Loop back mode (internal and external) ­ * - Transceiver standby mode
  7. Maskable interrupt
  8. Extended status and error report ­ * - Capturing of last occurred kind of error and of arbitration lost position ­ * - Programmable error warning limit
  9. Time-stamping ­ * - Supports ISO 11898-4 Time-Triggered CAN with partial hardware


配置设置 (sn34f78x_hal_conf.h)

  1. 如果需要使用CAN, 需要打开 configUSE_CAN_PERIPHERAL
  2. 如果要使用回调，需要打开 configUSE_CAN_REGISTER_CALLBACKS
