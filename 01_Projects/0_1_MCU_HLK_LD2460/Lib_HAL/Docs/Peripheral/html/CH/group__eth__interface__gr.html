Description

以太网是高质量的以太网控制器，配有 DMA 功能。它包含了 AHB/AXI wrapper , DMA 引擎, 单片存储器 (TXFIFO和RXFIFO), MAC, 和 MII/RMII/GMII/RGMII 接口。
为了降低 Host CPU 的处理负载，以太网执行 TCP, UDP, IPv4, 和 IPv6 校验和产出与验证，支持 VLAN tagging。
对于 QoS 和 CoS 需求，FTGMAC030 支持高优先级的队列为发送包降低 Host CPU 的处理负载。
同样会提供 Wake-On-LAN 功能。三个唤醒事件：Link状态更改，魔术包，唤醒帧。
该函数允许系统包含 FTGMAC030，FTGMAC030 会从远程被唤醒。


Features

  1. DMA engine for transmitting and receiving packets
  2. Supports TCP, UDP, IPv4, and IPv6 checksum offloads
  3. Supports IEEE 802.1Q VLAN tag insertion and removal
  4. Supports high-priority queues for QoS and CoS applications
  5. Supports Wake-on-LAN function and three wake-up events: Link status change, magic packet, and wake-up frame
  6. Independent TX/RX FIFOs
  7. Supports half and full duplex operations
  8. Supports flow control for full duplex and back pressure for half duplex
  9. Supports MII/RMII interfaces
  10. Supports jumbo packets (9K bytes)
  11. Supports IEEE 1588v1 and 1588v2 frame recognition and timestamp generation
  12. Supports IEEE 802.3az LPI assertion and detection


配置设置 (sn34f78x_hal_conf.h)

  1. 需要打开 configUSE_ETH_PERIPHERAL
  2. 如果要使用回调，需要打开 configUSE_ETH_REGISTER_CALLBACKS


信号线

信号 名字 说明
TX_CK Transmit Clock 发送时钟
TXD[3:0] Transmit Data bit 0~3 0~3数据位，用于发送
TX_EN Transmit enable 对于 MII 节点，使能发送
TX_ER Transmit error 发送错误
RX_CK Receive Clock 接收时钟
RXD[3:0] Receive Data bit 0~3 0~3数据位，用于接收
RX_EN Receive enable 对于 MII 节点，使能接收
RX_ER Receive error 接收错误
RX_DV Receive data valid 接收对于 MII 节点有效的数据
CRS Carrier sense 载波侦听，用于发送
COL Collision detect 载波侦听，用于发送
