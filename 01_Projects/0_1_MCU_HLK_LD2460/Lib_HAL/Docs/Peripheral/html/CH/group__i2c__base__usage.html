Base Usage
I2C Interface » Usage Flow

Description

在基本使用中，I2C有四种传输模式： Blocking ，Interrupt ，Burst 以及 Sequence 模式

Functions

特点 功能 说明 注意事项
Blocking mode Master 发送 通过 HAL_I2C_Master_Transmit API
作为 Master, 在 Blocking 模式下发送一定数量的数据 		超时设置不应太短。
Master 接收 通过 HAL_I2C_Master_Receive API
作为 Master, 在 Blocking 模式下接收数据 		超时设置不应太短
Slave 发送 通过 HAL_I2C_Slave_Transmit API
作为 Slave, 在 Blocking 模式下发送一定数量的数据 		超时设置不应太短
Slave 接收 通过 HAL_I2C_Slave_Receive API
作为 Slave, 在 Blocking 模式下接收数据 		超时设置不应太短
Interrupt mode Master 发送 通过 HAL_I2C_Master_Transmit_IT API
作为 Master，通过中断异步发送一定数量的数据 		超时设置不应太短
Master 接收 通过 HAL_I2C_Master_Receive_IT API
作为 Master，通过中断异步接收数据 		传输完成的时候，会产生中断
Slave 发送 通过 HAL_I2C_Slave_Transmit_IT API
作为 Slave，通过中断异步发送一定数量的数据 		传输完成的时候，会产生中断
Slave 接收 通过 HAL_I2C_Slave_Receive_IT API
作为 Slave，通过中断异步接收数据 		传输完成的时候，会产生中断
Sequential mode Master 发送 通过 HAL_I2C_Master_Seq_Transmit_IT API
作为 Master，在 Sequence 模式下异步发送一定数量的数据 		传输完成的时候，会产生中断
Master 接收 通过 HAL_I2C_Master_Seq_Receive_IT API
作为 Master，在 Sequence 模式下异步接收数据 		传输完成的时候，会产生中断
Slave 接收 通过 HAL_I2C_Slave_Seq_Transmit_IT API
作为 Slave，在 Sequence 模式下异步发送一定数量的数据 		传输完成的时候，会产生中断
Slave receive 通过 HAL_I2C_Slave_Seq_Receive_IT API
作为 Slave，在 Sequence 模式下异步接收数据 		传输完成的时候，会产生中断
Burst mode Master 发送 通过 HAL_I2C_Master_Transmit_BURST API
作为 Master，在 Burst 模式下异步发送一定数量的数据 		传输完成的时候，会产生中断
Master 接收 通过 HAL_I2C_Master_Receive_BURST API
作为 Master在，Burst 模式下异步接收一定数量的数据 		传输完成的时候，会产生中断
Abort Master 中止 通过 HAL_I2C_Master_Abort_IT API
中止 Master IT 模式下正在进行的传输 		只有 Master 可以中止传输
