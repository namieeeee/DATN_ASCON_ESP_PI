Description

FileManager 调试模块，基于FATFS进行进一步封装，便于操作不同类型文件；目前支持对 bin, txt, wav, mp3, A32, bmp, jpg, gif文件操作


  • 特点
  1. 基于FATFS, 对不同文件类型进行统一管理
  2. 支持对片外介质的 读/写, 分区, 写文件系统等操作; 片外介质目前包括 SD, 片外Flash;
  3. 提供两套API, 一种针对 文件 进行操作，一种针对 文件夹 进行操作


  • 配置设置 (sn34f78x_hal_conf.h)
  1. 需要设置 configUSE_FILEMANAGER_MIDDLEWARE = 1
  2. 可以根据需要在 sn34f78x_hal_conf.h 中选择一种或多种文件类型：
    a. 如果操作文件类型为 BIN，则需要设定 configUSE_BIN_FILE = 1
    b. 如果操作文件类型为 TXT，则需要设定 configUSE_TXT_FILE = 1
    c. 如果操作文件类型为 WAV，则需要设定 configUSE_WAV_FILE = 1
    d. 如果操作文件类型为 MP3，则需要设定 configUSE_MP3_FILE = 1
    e. 如果操作文件类型为 A32，则需要设定 configUSE_A32_FILE = 1
    f. 如果操作文件类型为 BMP，则需要设定 configUSE_BMP_FILE = 1
    g. 如果操作文件类型为 JPG，则需要设定 configUSE_JPG_FILE = 1
    h. 如果操作文件类型为 GIF，则需要设定 configUSE_GIF_FILE = 1
  3. 需要与Thirdparty中的FATFS和FreeRTOS同时使用，存储介质（Disk）和文件系统（File System）相关配置在FATFS的ffconf.h中
    a. USE_SD_AS_DISK: 支持SD卡作为存储介质
    b. USE_FLASH_AS_DISK: 支持Flash作为存储介质

  • 功能
    Filemanager 支持下面的功能：
    功能 说明 注意事项
    创建/读/写 文件(夹) 对支持的文件类型均可操作 a. 读文件时采用 HAL_MFile_ReadFile API, 该API调用一次将从目标文件读出一次buffer size的内容, 需要根据文件大小进行多次调用;
    b. 写文件时采用 HAL_MFile_WriteFile API, 该API调用一次将写入一次buffer size的内容, 需要根据文件大小进行多次调用;
    c. 对 文件 和 文件夹 的操作是两套不同API
    创建文件系统 若介质没有文件系统, 可在开发板上进行创建 需要 设置 configUSE_FILE_MAKE_FS = 1
    介质分区管理 可将介质分成盘符不同的区域 同一个介质最多分区 FF_VOLUMES 个, 需在 ffconf.h 内设置


  • 注意事项
  1. HAL_MFile_ReadFile 和 HAL_MFile_WriteFile API 需要根据文件大小进行多次调用;
  2. 文件名默认只支持英文, 若要支持其他语言, 需要结合 ffconf.h 中的配置进行修改;
