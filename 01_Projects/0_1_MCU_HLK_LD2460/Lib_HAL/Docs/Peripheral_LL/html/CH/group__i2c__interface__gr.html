Description

I2C ( I-squared-C, I-two-C, 或 IIC) is a multi-master bus and can be controlled by more than one bus master connected to it。

I2C key feature：

  • Only two buses are needed
  • I2C is a true multihost bus. All components have a simple Master/Slave relationship with each other
  • No baud rate is required; the Master device determines the bus clock
  • Each connected device can be addressed by a unique address
  • Arbitration and conflict detection mechanisms are provided

For more information about I2C, please refer to the following webpage：

  • WIKI： I2C
  • www.i2c-bus.org.


Features

  1. Standard I2C-compliant bus interfaces may be configured as Master or Slave
  2. I2C Master features：
    • Clock generation
    • Start and Stop generation
  3. I2C Slave features：
    • Programmable I2C Address
    • Stop bit detection
  4. Supports different communication speeds：
    • Standard Speed (up to 100KHz)
    • Fast Speed (up to 400 KHz)
    • Fast Plus (up to 1MHz)
    • High Speed (up to 3MHz)
  5. Arbitration is handled between simultaneously transmitting masters without corruption of serial data on the bus
  6. Programmable clock allows adjustment of I2C transfer rates。
  7. Data transfer is bidirectional between masters and slaves
  8. Serial clock synchronization allows devices with different bit rates to communicate via one serial bus
  9. Serial clock synchronization is used as a handshake mechanism to suspend and resume serial transfer.
  10. Generation and detection of 7-bit/10-bit addressing and General Call
  11. Supports glitch suppression
  12. Support Burst Transmission, up to 34 data transmission
