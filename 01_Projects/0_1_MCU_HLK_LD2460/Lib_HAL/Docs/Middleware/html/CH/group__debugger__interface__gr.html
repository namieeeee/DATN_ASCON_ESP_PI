Description

Debugger 调试模块，对 SWO, UART, USB CDC 功能模块化，使用简单，便于调试； 用于程序 Run-Time 时，通过 printf 输出 log 信息，同时为用户定义了8个级别来处理不同的状态 Debugger 模块, 提供 AT+CMD Console 功能，即输入命令实现对应的功能


  • 特点
  1. 支持 SWO, UART, USB CDC 调试打印
  2. 支持 AT+CMD 控制台控制
  3. 支持通过 get_char 获取输入信息
  4. 为用户定义了8个级别来处理不同的状态


  • 配置设置 (sn34f78x_hal_conf.h)
  1. 需要打开 configUSE_DEBUGGER_MIDDLEWARE
  2. 如果要使用 UART 打印模式，需要设定 configUSE_DEBUGGER_OPTION = 0
  3. 如果要使用 UART ATCMD 打印模式，需要设定 configUSE_DEBUGGER_OPTION = 1
  4. 如果要使用 SWO 打印模式， 需要设定 configUSE_DEBUGGER_OPTION = 2
  5. 如果要使用 USB CDC 打印模式， 需要设定 configUSE_DEBUGGER_OPTION = 3


  • 功能
    Debugger 支持下面的功能：
    模式 功能 说明 注意事项
    UART 打印模式 UART 通过 printf_uart
    实现 UART 串口输出字符     		UART输出存在乱码，先检查波特率是否匹配 或 系统时钟频率低
    通过 get_chars API

    实现SWO 输出字符串     		SWO 功能主要通过Keil IDE 读取输出字符数据

Debugger print

同样， Debugger 模块为用户定义了 8 个级别，以应对不同的状况。详细的列表如下所示：

项目 级别 说明
EMERG_PRINT configUSE_DEBUGGER_LEVEL >=0

用于紧急消息，通常发生在 Crash 之前的相关消息

ALERT_PRINT configUSE_DEBUGGER_LEVEL >=1 需要马上处理的情形
CRIT_PRINT configUSE_DEBUGGER_LEVEL >=2 危急情形，经常和严重的硬件或者软件问题有关
ERR_PRINT configUSE_DEBUGGER_LEVEL >=3 用于报告错误情形；设备驱动经常使用 KERN_ERR 报告硬件问题
WARN_PRINT configUSE_DEBUGGER_LEVEL >=4 有问题的情形的警告，这些问题本身不会造成系统的严重问题
NOTE_PRINT configUSE_DEBUGGER_LEVEL >=5 正常情形，但依然值得重视。在这个级别会报告一系列和安全有关的情形
INFO_PRINT configUSE_DEBUGGER_LEVEL >=6 信息类的消息。在这个级别，会在开启的时候，许多驱动会打印有关找到的硬件的信息
DEBUG_PRINT configUSE_DEBUGGER_LEVEL >=7 用于调试消息

configUSE_DEBUGGER_LEVEL 在 HAL_Common.h 中定义 上面的 Debugger 级别会和 printf 符合一起用
应在 Keil IDE C/C++ 页面定义 printf 符号, 相关内容可以通过 printf_uart, 或者 printf_swo 或者 printf_vcp 打印 如果 QS 创建了工程， printf 符号会由 QS 自动化产出

Mule Debugger USB CDC :

在 DEBUGGER_MODE_USB_CDC 模式，通过USB CDC接口类实现调试功能,USBD支持注册多个类，USBD对用户开放,在使用时需要先进行USB初始化，再初始化Debugger

  1. USBD初始化、注册CDC Class, 设置USBD 速率, USBD 启动； 说明，该功能需要USER进行配置.
  2. 初始化Debugger 模块
  3. 正常使用TX/RX 功能 注意：开启USBD USB_HS_IRQn 的中断，添加需要的pcd驱动，USBD 模块, 移植/实现CDC Class 文件. 详细请参考：Debugger/print_vcp.c

Debugger AT+CMD :

在 DEBUGGER_MODE_UART 模式，用户可以通过 CMD 任务监测命令行字符，以此执行特殊的功能。命令行格式为："AT+CMD=para", 其中：

  1. "AT+": 命令行函数固定的开头，表示输入的字符串是一个命令行
  2. "CMD": 函数的名字字符串
  3. "=para": 执行的函数不具有参数的时候，子字符串不存在。有参数的时候， '=' 后面会跟着参数。 "para" 是函数的参数字符串，可以被 ',' 分开

  4. 命令行的结尾由 Debugger_INIT_T 的 cmd_end 字段定义

    Debugger 有三个内置的命令：

  1. "AT+Dump=address,size": 该命令会在内存打印数据， "address" 作为开始位置， "size" 作为字节长度
  2. "AT+Update=address,value": 该命令会去修改内存地址值对应的值

  3. "AT+TaskInfo=name": 该命令会打印和当前设定的任务名相同的任务的相关信息，打印信息包含了状态，优先级等。如果未找到对应任务名，会打印 "No task is named"

    Debugger 支持外部的命令：

  • 1. 通过HAL_DBG_Reg_Cmd 注册 CMD 函数
  • 2. 获取 AT CMD 函数 HAL_DBG_ReceiveCmd , 根据 cmd_id 调用对应的函数。请参考控制台示例
  • 3. 在注册 CMD 名后，输入命令行，然后实现对应的函数

命令行字符串应小于 Debugger_Handle_T 中的 Debugger_INIT_T 的buff_size。
